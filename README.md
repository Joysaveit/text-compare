# Joy Comparison

一个用于文本与表格差异比对的 Chrome 扩展（也可直接本地打开页面使用）。

## 当前功能（与代码一致）

### 1. 文本对比
- 双栏输入，双栏结果展示（原始文本 / 修改文本）。
- 字符级差异高亮（删除 / 新增）。
- 差异导航：`上一处差异`、`下一处差异`、下拉定位、序号跳转。

### 2. 列表序号能力（文本模式）
- 可选开关：`忽略列表序号差异`。
- 支持一级与二级有序列表的语义对齐（例如 `1.` / `（1）` / `1.1` 等常见形式）。
- 开启忽略后：尽量忽略“序号格式差异”，聚焦正文差异。
- 关闭忽略后：序号差异也会计入比较结果。
- 同时提供列表编号校验摘要：漏号、跳号、重复、回退。
- 校验异常时，对异常编号前缀进行单独字体着色（不再整行着色）。

### 3. 表格对比
- 独立“表格对比”模式。
- 支持粘贴表格文本并做新增/删除/修改统计与渲染。
- 包含列级别差异展示（在匹配逻辑可识别时）。

### 4. 其他能力
- 主题切换（多套主题）。
- 历史记录：保存、查看、清空。
- 基础状态提示与错误提示。

## 使用方式

### 方式 A：作为 Chrome 扩展使用（推荐）
1. 打开 `chrome://extensions/`
2. 开启“开发者模式”
3. 选择“加载已解压的扩展程序”
4. 选择本目录 `compare-master`

### 方式 B：直接本地打开
- 直接打开 `index.html`
- 注意：部分扩展相关行为在普通网页环境与扩展环境会有差异

## 主要文件

- `index.html`：页面结构
- `style.css`：样式与主题
- `script.js`：核心逻辑（文本/表格对比、历史、主题）
- `manifest.json`：Chrome 扩展配置
- `background.js`：扩展后台脚本

## 已知问题（当前仍存在）

1. `script.js` 中 `TableParser.parseLine()` 会调用 `parseCSVLine()`，但该方法未实现。  
   影响：某些 CSV / 分号 / 竖线分隔输入在表格模式可能报错。

2. `script.js` 的历史恢复流程中仍有 `document.getElementById('compare')` 调用。  
   影响：部分历史记录“加载并比较”动作可能失效（页面并无 `#compare` 按钮）。

3. `script.js` 的表格 diff 仍使用整行值做 key。  
   影响：重复行可能被覆盖；“修改行”识别在某些场景会退化为“删除+新增”。

4. `test_extension.html` 的 `fileExists()` 目前固定返回 `true`。  
   影响：测试页不能真实发现缺失文件，只能作为演示页参考。
